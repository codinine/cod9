1. 웹의 성장 과정  
1990년 - WWW(World Wide Web)의 탄생  
1993년 - NCSA MOSAIC (그래픽 웹 브라우저)  
         하이퍼 텍스트 기반의 웹 브라우저 탄생  
1994년 - 넷스케이프, 아마존 창업 (최초의 인터넷 서점)  
1995년 - Windows 95 판매  
1996년 - Apple이 NeXT 인수  
1998년 - 구글 창업, AOL이 넷스케이프 인수  
1999년 - i-Mode 등장, 닷컴 버블  
2001년 - 닷컴 버블 붕괴  
2004년 - 페이스북 창업  
2007년 - 아이폰 출시  
2011년 - 애플 시가총액 세계 1위, 스티브잡스 사망  

2. 웹의 근간을 이루는 기술  
URI : 데이터는 어디에 있는가? (Uniform Resource Identifier)  
서버가 응용프로그램을 클라이언트(요청)에 전달  
URN과 URL을 포함하는 개념  

HTTP: 데이터는 어떻게 손에 넣는가? (Hyper Text Transper Protocol)  
URI에 지정된 데이터를 HTTP라는 규약을 거쳐 가져온다.  

HTML : 데이터는 어떻게 표현하는가? (Hyper Text MArkup Language)  
웹브라우저가 해석하는 언어  
마크업 언어 -> 구조를 표시, 태그를 이용  
HTML은 프로그래밍을 위한 언어가 아니다.  
HTML 내부언어, CSS ,Javascript 외부언어  

3. 웹 애플리케이션과 서버  
웹 애플리케이션(웹 서비스)  
웹 브라우저를 통해 HTTP로 접근해서 사용하는 응용프로그램  
검색엔진, 웹메일, 그룹웨어, 고객관리, ERP 등.  

정적 컨텐츠 : 텍스트, 이미지, 동영상 등.  
동적 컨텐츠 : 장바구니, 위치기반 등.  
정적+동적 컨텐츠 : 쇼핑몰 상품정보 + 주문현황,포인트,장바구니  

웹 서버 : 클라이언트가 요청한 컨텐츠를 탐색하여 응답, 응답내용 기록(log)  
애플리케이션 서버 : 웹 서버가 동적 컨텐츠 요청, 요청 내용 확인, 웹서버에 컨텐츠 전달  
DB 서버 : 애플리케이션이 데이터 조회 요청, 데이터 조회 결과 전달  
프록시 서버(대리, 대행) : 데이터를 대신 전달해줌  
 - 리버스 프록시 서버 : 부하 정도를 파악하여 응답받을 어플리케이션 서버 선택  
 - 포워드 프록시 서버 : 캐싱, 미리 저장한 데이터로 응답해주는 기능  
로드 밸런서 : 클라이언트와 웹서버 사이에서 스케쥴 관리
