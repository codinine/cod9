#### [](#같은-타입의-데이터가-많아진다면… "같은 타입의 데이터가 많아진다면…")같은 타입의 데이터가 많아진다면…

비슷한 데이터 여러개를 저장해야될 경우가 있다.  
예를 들어 한 반의 학생들의 이름, 학생들의 수학 성적처럼 말이다.  
이런식으로 같은 타입의 데이터들이 연속적으로 저장해야할때 우리는 `배열`이라는 것을  
사용할 수 있다. 배열에 저장되는 데이터들의 공간은 연속적으로 존재하여야만 한다.  
서랍장이나 급식판처럼 붙어있다고 보면 된다.  
배열명을 선언하는 것도 변수명을 선언하는 방식과 동일하다.  
다만 배열 뒤에는 `배열명(배열의갯수)` or `배열명[배열의갯수]` 형태로 선언한다.

#### [](#배열의-요소-번호 "배열의 요소 번호")배열의 요소 번호

앞서 말한바와 같이 배열의 같은 타입의 데이터들이 들어가야하고  
한 공간에는 하나의 데이터만 저장되어야 한다.  
배열이 연속적으로 존재함으로써 순서대로 `요소 번호`라는 것을 갖게 된다.  
그냥 배열에 순서가 있다고 생각하면 쉽다.

그냥 줄 번호를 셀때 보통 1부터 세지만 배열은 일반적으로 0부터 센다.  
이 `요소 번호`를 다른 말로 `인덱스`라고도 부른다.

우리는 데이터를 찾을때 이 요소번호를 활용하면  
쉽게 데이터를 찾을 수 있다.

배열명[3] -> 배열의 4번째에 저장된 데이터를 의미한다.(요소가 0부터 시작하므로)

#### [](#배열을-쓰는-이유 "배열을 쓰는 이유")배열을 쓰는 이유

배열을 쓰는 이유는 처음에 이야기 했던 것처럼 비슷한 데이터를 여러개 저장해야하기  
때문이다. 반대로 배열이 없다면 저 여러개의 데이터를 우리를 각각의 변수명을 선언하여  
저장해야 한다. 만약 저장해야될 데이터가 100개? 아니 1000개라면?  
배열은 배열명[100], 배열명[1000]이라고 하면 끝난다.

#### [](#2차원-배열 "2차원 배열")2차원 배열

배열이 꼭 한 줄 형태로만 존재하는 것은 아니다.  
일반적인 배열이 캐비닛 형태라면 2차원 배열은 호텔같은 형태이다.  
3차원 배열은 큐브같은 형태라고 생각하면 된다.  
3차원 배열까지 쓰는 경우는 흔하지 않으므로 2차원 배열을 조금 더 알아보자.

2차원 배열은 마치 같은 모양의 블럭을 옆으로 그리고 위로 쌓는 형태와 같다.  
예를 들면  
ex)  
[홍길동][국어점수]  
[홍길동][수학점수]  
[성춘향][국어점수]  
[성춘향][수학점수]  
[이몽룡][국어점수]  
[이몽룡][수학점수]

한쪽으로 학생들의 이름을 저장하고 위로는 각 학생들의 과목별 점수를 저장하게되면  
이것이 2차원 배열이 된다.  
위의 데이터가 순서대로 저장되었다면 ‘요소 번호’로 데이터를 찾아보자.  
배열명[행 요소 번호][열 요소 번호]  
성춘향의 국어점수를 찾고 싶다면 `배열명[1][0]`으로 찾을 수 있다.

#### [](#문자열도-배열이다 "문자열도 배열이다.")문자열도 배열이다.

어떤 의미에선 문자열도 문자들이 저장된 배열이라고 할 수 있다.  
컴퓨터는 문자열을 각각의 문자로 쪼개어 숫자로 변환하여 저장한다.  
가장 흔히 쓰이는 이 변환 방식을 `아스키 코드`라고 하는데,  
대문자 ‘A’가 `65`로 시작되어 순차적으로 저장된다.

만약 ‘ABC’라는 문자열을 저장해야 한다면 컴퓨터는 이것을  
[0]번 상자에 ‘65’  
[1]번 상자에 ‘66’  
[2]번 상자에 ‘67’을 저장하는 것이다.

컴퓨터는 이러한 문자열의 길이를 파악하기 위해 `보초 값`이라는 것을 사용하는데  
문자열 끝에 0 또는 null 값을 저장하여 문자열을 끝을 확인한다.  
사용자가 직접 길이을 지정하여 사용하는 방법도 있다.

#### [](#관용적인-변수명 "관용적인 변수명")관용적인 변수명

*   반복문 - i,j,k
*   배열의 첨자 - index, idx
*   숫를 세는 변수 - count, counter, cnt
*   문자열 - str, string
