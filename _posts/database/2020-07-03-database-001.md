### 네트워크
##### HTTP (HyperText Transfer Protocol)  
 - 웹에서 클라이언트와 서버 사이에 정보를 주고 받는 프로토콜
 - TCP와 UDP 사용
 - 주로 80번 포트 사용 (절대적이진 않음)

##### HTTPS (HTTP +  Secure Socket Layer)
 - HTTP의 보안이 강화된 프로토콜
 - SSL이나 TLS 프로토콜을 통해 세션 데이터를 암호화
 - 주로 443 포트 사용 (절대적이진 않음)

##### TCP와 UDP
종류 |TCP | UDP |
---|---|---|
명칭|Transmission Control Protocol<br>전송제어 프로토콜 |User Datagram Protocol<br>사용자 데이터그램 프로토콜|
연결방식|연결형|비연결형|
패킷교환방식|가상회선|데이터그램|
전송순서|순서보장|순서바뀔수있음|
수신여부|확인|확인안함|
통신방식|1:1|1:1,1:N,N:N|
신뢰성|높다|낮다|
속도|느리다|빠르다|

* 간단정리 : TCP 느리지만 안정적, UDP 빠르지만 불안정

##### 웹의 동작 원리
1. 웹 브라우저를 통해 URI 입력
2. URI가 도메인이면 DNS 서버에서 검색
3. DNS 서버에서 해당 도메인의 IP를 찾아 HTTP를 통해 요청 메세지 생성
4. 요청 메세지는 TCP를 이용해 해당 IP의 웹서버에 전송
5. 웹서버는 해당 URL의 정보를 찾아 HTTP 응답 메세지 생성
6. TCP를 이용하여 클라이언트에게 전송
7. 웹 브라우저를 응답받은 정보를 해석하여 출력

##### GET VS POST
 - GET : URL 뒤에 데이터 전달, 보안 위험, 길이제한, 캐싱O
 - POST : Body에 데이터 전달, 기본 보안, 길이제한X, 캐싱X

##### 쿠키와 세션
 - 쿠키 : PC에 저장, 브라우저를 종료해도 남아 있음, ex) 로그인 패스워드 저장
 - 세션 : 브라우저에 저장, 브라우저가 종료되면 사라짐, ex) 로그인 상태

### Javascript
##### var VS let VS const
 - var : 함수레벨 스코프
 - let : 블록레벨 스코프,
 - const : 블록레벨 스코프, 상수개념(값을 한번 할당하면 변경 불가)

##### 동기와 비동기
 - 동기 : 기다렸다가 순서대로 처리
 - 비동기 : 기다리지 않고 완료되면 처리
 - 콜백함수 : 비동기, 함수 중첩으로 콜백지옥 발생
 - promise : 비동기, 콜백 대체, 동기형태 코드
 - async : function 앞에 사용, 비동기 함수임을 명시
 - await : async 함수 안에서 사용, 비동기 처리할 함수 호출 또는 fetch로 URL 응답 요청
